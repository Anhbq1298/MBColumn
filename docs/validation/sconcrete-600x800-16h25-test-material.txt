Note on S-CONCRETE sign convention
----------------------------------
The exported S-CONCRETE N-vs-M table reports compression using negative axial load values. MBColumn internal convention uses compression positive. Therefore, convert signs consistently before comparing:

S-CONCRETE axial compression value -12113.26 kN should map to MBColumn compression-positive value +12113.26 kN if MBColumn output follows the repository convention.

Comparison Procedure
--------------------
1. Run MBColumn EC2 fiber PMM solver for the section and reinforcement above.
2. Disable slenderness, second-order effects, and minimum eccentricity if those options exist in the comparison workflow.
3. Export MBColumn N-M curves at the same theta values listed above.
4. Convert sign conventions before comparison.
5. Compare moment capacity at matching axial force levels.
6. Report absolute and percentage difference.

Tolerance
---------
PMM ratio or capacity difference:
- <= 3 percent: good
- 3 to 5 percent: acceptable with explanation
- > 5 percent: investigate

Possible difference sources
---------------------------
- S-CONCRETE material law versus MBColumn EC2 material law
- concrete stress block versus fiber integration
- concrete fiber mesh density
- main bar coordinate interpretation behind H10 links
- sign convention
- moment direction convention
- minimum eccentricity settings
- second-order/slenderness effects
- hidden commercial software assumptions
